## PR Order

1. Open one PR first, preferably to `filipecalegario/awesome-vibe-coding`.
2. Wait for maintainer feedback before sending the same project to multiple
   awesome lists.
3. Adjust wording based on feedback.
4. Submit to `ai-for-developers/awesome-ai-coding-tools`.
5. Submit to the remaining lists only after the README and Korean/English launch
   posts have public feedback.

This avoids looking like bulk self-promotion and gives maintainers a clearer
reason to include the project.
