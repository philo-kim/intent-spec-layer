# Contributing

Contributions are welcome when they improve clarity, evidence, templates, or
real-world applicability.

## Good Contributions

- A clearer rule for choosing L0/L1/L2/L3.
- A domain example that exposes a missing failure mode.
- A reproducible experiment that compares approaches.
- A template improvement that reduces ambiguity.
- A correction to a source, claim, or reference.

## Pull Request Expectations

1. Explain what changed and why.
2. Link affected guide, template, example, or experiment files.
3. Run `npm run check` when experiment code changes.
4. Run `npm run req:test:generate` when adding or changing requirements.
5. Avoid adding tool-specific assumptions unless the guide clearly marks them as
   tool-specific.

## Method Standard

This project values falsifiable claims. If a change makes a broad claim, include
one of:

- a reproducible experiment;
- a concrete field example;
- a well-scoped limitation;
- a primary source link.
